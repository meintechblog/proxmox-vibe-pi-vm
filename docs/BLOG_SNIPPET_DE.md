# Blog-/Video-Snippet (DE)

## Kurzfassung für den Blog

Für mein Vibe-Coding-Setup auf Proxmox nutze ich eine schlanke, reproduzierbare x86-VM mit Ubuntu Server (headless). Das Setup ist bewusst auf schnellen Terminal-Workflow ausgelegt: VS Code Remote SSH, AI-Coding (z. B. Codex), und API-/Netzwerkprojekte wie Ulanzi-Display-Integrationen.

Wichtig: Das ist keine 1:1 Raspberry-Pi-Hardware-Emulation, sondern ein pragmatisches Dev-Setup mit sehr ähnlichem Linux-Workflow. Für die meisten Coding-/Automation-Use-Cases ist das perfekt. Für GPIO/ARM-spezifische Themen sollte man final auf echter Pi-Hardware gegenchecken.

👉 Repo (öffentlich): https://github.com/hulki-bot/proxmox-vibe-pi-vm
